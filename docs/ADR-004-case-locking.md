# ADR-004: نظام الأقفال متعددة المتطوعين

**الحالة**: معتمد | **التاريخ**: 2024  
**السياق**: عدة متطوعين يتصفحون المهام في وقت واحد، احتمال توزيع مزدوج لنفس الأسرة.

**الخيارات**: Pessimistic Locking (قفل قبلي), Optimistic + Timeout (قفل بعدي بمهلة), بدون قفل.

**القرار**: Optimistic Locking مع جدول `case_locks` ومهلة 30 دقيقة.

**آلية العمل**:
1. `acquireLock(familyId, campaignId, volunteerId)` ← يحاول INSERT
2. إذا نجح → المتطوع يملك القفل (صلاحية 30 دقيقة)
3. إذا فشل (duplicate key) → `{ lockedBy: "أحمد" }`
4. عند تسجيل `transactions` → Trigger يحرر القفل تلقائياً
5. `cleanup_expired_locks()` → ينظف الأقفال المنتهية

**المخاطر**:
- سباق (Race condition) بين cleanup و acquire — مقبول في سيناريو الاستخدام
- متطوع قد ينسى ويترك القفل منتهياً

**التحسينات المستقبلية**:
- إشعار للمتطوع باقتراب انتهاء القفل
- إظهار "مُحرر بواسطة" مع اسم المتطوع الحالي
