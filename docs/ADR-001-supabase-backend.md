# ADR-001: Supabase كـ Backend موحد

**الحالة**: معتمد | **التاريخ**: 2024  
**السياق**: الحاجة إلى قاعدة بيانات علائقية مع مصادقة وصلاحيات.

**الخيارات**: Firebase (NoSQL), Supabase (PostgreSQL), Backend مخصص (Node.js).

**القرار**: Supabase. PostgreSQL + Auth + RLS يغطي كل الاحتياجات دون تطوير Backend منفصل.

**الإيجابيات**:
- تطوير أسرع (لا حاجة لكتابة REST API من الصفر)
- RLS يضمن أماناً على مستوى الصف
- Realtime subscriptions للأقفال والتحديثات

**السلبيات**:
- Vendor lock-in (Supabase)
- استعلامات معقدة قد تكون أبطأ مع RLS
- الاعتماد على توفر الخدمة

**البدائل الملغاة**:
- Firebase: NoSQL لا يناسب العلاقات المعقدة (أسر ← أطفال ← حملات ← معاملات)
- Node.js + Express: تكلفة تطوير وصيانة أعلى
