# ADR-003: محرك الأولويات متعدد العوامل

**الحالة**: معتمد | **التاريخ**: 2024  
**السياق**: تحتاج الجمعية لتحديد أولويات الأسر المستحقة بناءً على معايير متعددة.

**الخيارات**: حساب يدوي للمدير، خوارزمية أوتوماتيكية، نظام ترجيح مخصص.

**القرار**: محرك أولويات بنظام النقاط (Point-based) مكرر في Frontend (`priorityEngine.ts`) و Backend (`calculate_priority_score` في PostgreSQL).

**النمط**:
| المعيار | النقاط القصوى |
|---------|--------------|
| أرملة | 25 |
| مطلقة | 15 |
| مرض مزمن | 15 |
| إعاقة | 20 |
| أطفال أيتام | 20 |
| عدد الأطفال (×5) | 25 |
| درجة الهشاشة (±10) | 10 |

**المستويات**: حرجة (≥70) ← عالية (≥45) ← متوسطة (≥20) ← منخفضة (<20)

**الإيجابيات**:
- دمج الـ Score مع RLS للتوجيه الذكي
- الحساب موجود في مكانين: لضمان الاتساق

**السلبيات**:
- التكرار بين TS و SQL — أي تعديل يحتاج تغييراً في مكانين
